Client-Side Web Development Tools

Modern web toolings provide developers with ways to speed up the development process by automating low-level code generation. In other words, web tooling writes HTML, CSS, and JavaScript for you, so you can spend more time on application and web design.

MDN groups client-side web tools into three categories:

  • Safety net - used in the development process
  • Transformation - used to transform code, for example from some intermediate language into JavaScript
  • Post-development - used for things like testing and deployment

Let's break down each category in a bit more detail.

Safety Net Tools

Safety net tools are development tools that help keep your code well-formatted, traceable, and neat. Safety net tools include,

  • Linters such as,
    • ESlint, the standard for JavaScript
    • CSS Lint
    • webhint for web development best practices and accessibility
  • Version control systems (VCS), of which Git is commonly used for web dev
  • Code formatters such as Prettier
  • Bundlers and Packagers which do things like tree-shake and minify your code. Examples include Parcel and webpack.

Transformation Tools

Transformation tools include things like transpilers and polyfills, which allow you to use modern JavaScript, CSS, and HTML in browsers which may not yet support newer features in a language. Interpretation of other languages, such as TypeScript, into one of the basic web languages is another transformation.

Tools which allow you to use the latest JavaScript and CSS include,

  • Babel, a JavaScript compiler
  • PostCSS, a CSS transformer

Web frameworks and superset languages include transformers to render code as JavaScript, CSS, or HTML. Examples include,

  • Sass
  • TypeScript
  • Frameworks such as React

Post-Development Tools

Post-development tools are mainly automatic tools that keep the code running online. They include,

  • Testing frameworks such as Jest and Mocha
  • Automated test tools such as Travis CI and Jenkins. You'll notice that these are usually CI/CD tools
  • Deployment tools such as Netlify and Vercel which will automatically deploy changes to a production site, typically after a repo push and automatic testing
