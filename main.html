Client-Side Web Development Tools

Modern web toolings provide developers with ways to speed up the development process by automating low-level code generation. In other words, web tooling writes HTML, CSS, and JavaScript for you, so you can spend more time on application and web design.

MDN groups client-side web tools into three categories:

  • Safety net - used in the development process
  • Transformation - used to transform code, for example from some intermediate language into JavaScript
  • Post-development - used for things like testing and deployment

Let's break down each category in a bit more detail.

Safety Net Tools

Safety net tools are development tools that help keep your code well-formatted, traceable, and neat. Safety net tools include,

  • Linters such as,
    • ESlint, the standard for JavaScript
    • CSS Lint
    • webhint for web development best practices and accessibility
  • Version control systems (VCS), of which Git is commonly used for web dev
  • Code formatters such as Prettier
  • Bundlers and Packagers which do things like tree-shake and minify your code. Examples include Parcel and webpack.
